In the process of building a framework to run the targetscope application, we want to build a smaller problem that can be used to start creating a web service that will accept requests, process dependencies and push responses.

We start with the larger problem described in the target gene-to-TFBS document:

Inputs
  1. Set of target gene transcription start sites (TSSs)
  2. Set of all correlated interactions (pairings of promoter regions and distal element regions)
Outputs
  1. Set of target gene TSSs, filtered on the gene name RARA (“RARA TSSs”)
  2. Set of all correlated interactions that are filtered on the set of RARA TSSs
Operations
  1. Filter operation that creates “RARA TSSs” set
  2. Map operation that finds all RARA TSSs that overlap interaction promoter regions

Here is a graphical overview of the pipeline:

One of the two outputs is in the middle of the pipeline. The web service will generate a list of intermediate dependencies and push them to the next step of analysis, as needed.
